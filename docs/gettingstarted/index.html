Getting Started

Usage Example

Companion must be only configured by his Companion Configuration class.

// Creates an empty companion configuration class
Companion *companion = new Companion();

This configuration class specified which image processing implementation should used. Following components must be initialized.

  • Callback handling
  • Search model
  • Image processing
  • Companion configuration

Callbacks

Simple set an result and error handler to an companion configuration class to obtain results and errors by an running companion execution.

companion->setResultHandler(resultHandler);
companion->setErrorHandler(errorHandler);

Result Handling

To obtain results from Companion an Result or Error Handler is needed to implement. An result handler must be specified as type:

void resultHandler(CALLBACK_RESULT results, cv::Mat source)

Type definition from CALLBACK_RESULT is an vector from Result:

#define CALLBACK_RESULT std::vector<Companion::Model::Result*>

For example results are all detected object from an 2D Feature Matching in an image frame.

void resultHandler(CALLBACK_RESULT results, cv::Mat source) {
    Companion::Model::Result *result;

    for(size_t i = 0; i < results.size(); i++) {

        // Mark the detected object
        result = results.at(i);
        result->getModel()->draw(source)
        ;

        // Draw the id of the detected object
        Companion::Draw::Frame *frame = dynamic_cast<Companion::Draw::Frame*>(result->getModel());
        cv::putText(source,
                    std::to_string(result->getId()),
                    frame->getTopRight(),
                    cv::FONT_HERSHEY_DUPLEX,
                    2,
                    frame->getColor(),
                    frame->getThickness());
    }

    cv::cvtColor(source, source, CV_RGB2BGR);
    cv::imshow("Object detection", source);
    cv::waitKey(1);
    source.release();
    results.clear();
}

Error Handling

If an error occurred by an running image processing. Comapnion will thrown an specific Companion::Error::Code. An specialized error message can be obtain from his Utility method.

void errorHandler(Companion::Error::Code code) {
    // Obtain detailed error message from code
    std::cout << Companion::Error::getError(code) << "\n";
}

Search Model

All search models are implemented from abstract ImageRecognitionModel class for each image processing. For this 2D object detection operation an FeatureMatchingModel must be created for all searched models.

Companion::Model::Processing::FeatureMatchingModel* model;
for (int i = 0; i < images.size(); i++) {

    model = new Companion::Model::Processing::FeatureMatchingModel();
    model->setID(i);
    model->setImage(cv::imread(images[i], cv::IMREAD_GRAYSCALE));

    if(!companion->addModel(model)) {
        std::cout << "Model not added";
    }
}

Image Processing

All supported image processing operations are implemented from ImageProcessing. For example a 2D Object Detection with usage from OpenCV 3 feature matching algorithms.

int type = cv::DescriptorMatcher::BRUTEFORCE_HAMMING;
cv::Ptr<cv::DescriptorMatcher> matcher = cv::DescriptorMatcher::create(type);
cv::Ptr<cv::BRISK> feature = cv::BRISK::create(60);

Companion::Algorithm::Matching::Matching *matching = new Companion::Algorithm::Matching::FeatureMatching(
  feature, feature, matcher, type, 10, 40, true);

Companion::Processing::FeatureDetection* detection = new Companion::Processing::FeatureDetection(
  matching, Companion::SCALING::SCALE_640x360);
  
companion->setProcessing(detection);

Execution

After all configuration steps from Companion are done execute this configuration class in a try/catch block. If Companion is missed configured a Companion Error Code will be thrown.

try {
    // Execute companion
    companion->run(ps);
} catch (Companion::Error::Code errorCode) {
    errorHandler(errorCode);
}
